--- layout: post title: 又找到一个wp的coding插件的问题 tags: - coding - Web status: publish type: post published: true comments: true --- 在 geshi/geshi.php 里面，有个函数会将 &,>,<," 这些替换为html字符。而同时呢，wp 的 wp-includes/formatting.php 里面也会将这些字符做处理，这样就混乱了。解决方法也很简单，把coding插件里面的相应函数注释掉就好了（目前发现注释掉&和"就可以）。 
        // init

        $aTransSpecchar = array(
//            '&' => '&',
//            '"' => '"',
            '<' => '<',
            '>' => '>' 
            );            

        $aTransSpecchar = array();
